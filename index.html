🦐 什么是 Skill？

为 OpenClaw 编写的「能力扩展包」，让它在特定领域变得专业

💡 一句话理解

Skill = 告诉 OpenClaw「遇到这种情况时，应该怎么做」的说明书

类比理解

🏥

通用医生

普通感冒能看，但不够专业

⚡

专业科室

有了「骨科 Skill」，就能看复杂的骨折

🎯

精确匹配

用户一说症状，就知道该用什么 Skill

真实例子

🎨 一个「小红书文案 Skill」长这样

当用户说「帮我写个小红书推广文案」时...

OpenClaw 看到关键词：「小红书」「文案」「推广」
    ↓
匹配到 wechat-copywriting Skill
    ↓
读取 Skill 里的专业指令，开始写作

🎯 思考一下

你的工作中，有哪些重复性的任务想要自动化？

比如：每次都要写固定格式的报告、分析数据、回复特定内容...

⚙️ Skill 的工作原理

理解触发机制，才能写出精准的 Skill

核心流程

用户请求
「帮我写公众号文案」
↓
OpenClaw 匹配
根据 description 判断用哪个 Skill
↓
加载 SKILL.md
读取指令和流程
↓
执行任务
按照 Skill 指引工作
↓
返回结果
完成用户的请求
🔑 关键点

description 写得好不好，决定了 Skill 能不能被正确触发！

渐进式加载（省 Token）

层级 内容 何时加载
1 name + description 始终加载（~100字）
2 SKILL.md 正文 Skill 被触发时（<5k字）
3 references/scripts 按需加载（不限量）

触发动画

📝 假设用户说：

「我的 PDF 文件怎么加水印？」

OpenClaw 扫描所有 Skill 的 description：

Skill A: "处理图片编辑..." ❌ 不匹配
Skill B: "当用户需要处理 PDF..." ✅ 匹配！
    ↓
加载 Skill B，执行 PDF 相关指令

📁 Skill 的目录结构

最简单只需要 1 个文件，复杂的有 4 个文件夹

最小结构

📂 my-first-skill/
📄 SKILL.md ← 必须有

完整结构

📂 skill-name/ ← 名字 = Skill 名字，全小写
📄 SKILL.md ← 必须：核心文件
📂 scripts/ ← 可选：可执行脚本
📄 rotate_pdf.py
📂 references/ ← 可选：参考资料
📄 api-docs.md
📂 assets/ ← 可选：资源文件
📄 logo.png

各文件夹作用

📄

SKILL.md

核心指令、工作流程、触发条件

🔧

scripts/

重复性代码脚本，稳定可靠

📚

references/

详细文档，按需加载，不占空间

🎨

assets/

输出时用的模板、图片等

⚠️ 命名规范

文件夹名 = Skill 名，全小写，用 - 连接
例如：wechat-copywriting、pdf-editor

📝 SKILL.md 详解

SKILL.md 是 Skill 的灵魂，分为两部分：元数据 + 正文

Part 1：YAML Frontmatter（元数据）

📋 必须包含的字段
---
name: skill的名称           # 英文小写+中划线
description: 这段描述决定何时触发这个 Skill！
            要写清楚：做什么 + 什么场景触发
            越具体越好，OpenClaw 靠它匹配
---
💡 description 写法技巧

✅ 好：「当用户需要创建、编辑 Word 文档（.docx）时使用」
❌ 差：「处理文档」

Part 2：Body（正文）

📖 正文结构建议
# Skill 名称

## 快速开始
简要说明如何使用

## 工作流程
1. 第一步
2. 第二步
3. 第三步

## 详细指南（按需）
- 进阶内容 → 链接到 references/
- 示例代码 → 直接写这里
- 注意事项 → 容易出错的地方

一个完整例子

---
name: wechat-copywriting
description: 辅助公众号运营和推文撰写。当用户提到：
              - 写公众号文章/推文/内容
              - 公众号选题/标题/文案
              - 微信图文消息
              使用此 Skill。
---

1. 了解账号定位和目标用户
2. 确定内容方向和选题
3. 撰写标题 + 正文
4. 给出发布建议

## 注意事项

- 开头要制造悬念或共鸣

📦 Bundled Resources

可选的资源文件夹，让 Skill 更强大

scripts/ - 可执行脚本

🤖

适用场景

重复性、稳定可靠的代码
比如：PDF旋转、图片处理

✨

优势

不占 token
执行结果稳定
可反复使用

references/ - 参考文档

📚 使用方式

详细内容不放 SKILL.md，放到这里

# SKILL.md（简洁版）
See [API文档](references/api.md) for details.

# references/api.md（详细内容）
# API 完整文档...

assets/ - 资源文件

📝

模板文件

报告模板、邮件模板

🎨

品牌素材

Logo、图片、图标

📦

项目模板

React/HTML 脚手架

📌 什么时候用？

scripts 代码要稳定重复运行
references 内容多、详细、不每次都要看
assets 输出时要引用文件

🔄 编写流程 6 步法

从需求到打包，手把手教你写 Skill

  1. 理解需求

    这个 Skill 要解决什么问题？用户会怎么说？

  2. 规划内容

    需要哪些 scripts / references / assets？

  3. 初始化

    用 init_skill.py 生成模板

  4. 编辑实现

    写 SKILL.md 和资源文件

  5. 打包

    用 package_skill.py 打包成 .skill 文件

  6. 测试迭代

    实际使用，发现问题继续改进

Step 3 初始化命令

scripts/init_skill.py <skill-name> --path <output-path> [--resources scripts,references,assets]

Step 5 打包命令

scripts/package_skill.py <path/to/skill-folder>

name: wechat-ops

              - 写公众号/推文/文章
              - 公众号选题/标题/内容
              - 微信运营/涨粉/互动
              使用此 Skill。
---

# 公众号运营助手

## 工作流程

1. **确认需求**
   - 账号定位是什么？
   - 目标用户是谁？
   - 这次要发什么类型的内容？

2. **内容规划**
   - 选题方向
   - 内容类型（干货/故事/热点/产品）
   - 参考：[references/content-types.md](references/content-types.md)

3. **撰写内容**
   - 标题：吸引点击，≤14字
   - 开头：制造悬念或共鸣
   - 正文：逻辑清晰，适当分段
   - 结尾：引导互动

4. **发布建议**
   - 最佳发布时间
   - 标签选择
   - 推广策略

## 标题模板

参考：[references/title-templates.md](references/title-templates.md)
